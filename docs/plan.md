# 감정 쓰레기통 앱 기획 (v0.1)

작성일: 2026-02-08

**목표**
- 로컬에서 간단히 감정 기록
- 해소되면 체크해서 해소 시각만 기록
- 추후 원격 DB(Postgres)로 확장 가능하게 설계

**핵심 플로우**
1. 어떤 일이 있었는지 입력
2. 그때 감정이 어땠는지 기록
3. 해소됨 체크 → 해소 시각 기록

**MVP 범위**
- 로컬 저장만 지원(추후 원격 DB(Postgres) 연결 예정)
- 감정 기록(상황/감정/시간)
- 해소됨 체크(해소 시각 기록)
- 목록/검색(간단 텍스트 검색)

**스키마(논리 모델, 저장 방식 독립)**

Entry
| 필드 | 타입 | 설명 |
| --- | --- | --- |
| id | string (uuid) | 항목 고유 ID |
| created_at | datetime | 생성 시각 |
| updated_at | datetime | 수정 시각 |
| event_text | string | 어떤 일이 있었는지 |
| emotion | enum | 감정 타입 |
| resolved_at | datetime | 해소 처리 시각(없으면 미해소) |

**감정 enum 목록(초안)**
- 화남
- 불안
- 슬픔
- 기쁨
- 짜증
- 두려움
- 피곤
- 무기력
